## E. DATA MANAGEMENT PLAN (DMP)

**Data Types & Sensitivity**:
- **Scientific data** (rasters, MCDA outputs): Public, CC-BY-4.0 license
- **Community survey data** (demographics, income, education): Anonymized, ethical review required
- **Financial data** (tariff collection, payment history): Sensitive, restricted access

**Archiving & Preservation**:
- **DOI assignment**: Zenodo.org (pending publication)
- **Versioning**: Git tags for each release (v0.1 Phase 1 validation, v1.0 peer-review, v1.1 post-field)
- **Storage**: 500 GB shared cloud (AWS S3 + redundant local backup)
- **Retention**: 10 years minimum (funder requirement), then decision on conversion to archival format

**Reproducibility**:
- **Docker image**: Guarantees exact Python 3.10 + dependencies reproducible on any machine
- **Notebook**: Jupyter outputs showing exact results (with random seed fixed)
- **Data URLs**: Permanent links to NASA POWER, Sentinel-2, SRTM (will outlive individual downloads)

## F. ETHICS & CONSENT PROTOCOL

**Research Ethics Approval**: Required from ISPTEC ethics committee + international collaborators (MIT, if involved)

**Informed Consent for Community Surveys**:
- Written consent form in Portuguese, distributed before any data collection
- Clearly explain: purpose (electricity planning), confidentiality (data anonymized), risks (minimal—just interview), benefits (priority for solar if eligible)
- Consent recorded (signature or verbal + witness)

**Data Privacy Safeguards**:
- Individual household IDs stripped; only zone + settlement type retained
- GPS coordinates generalized to 500 m grid (prevents re-identification)
- Names, phone numbers stored separately in encrypted database (access logged)
- Survey data retention: 3 years post-publication, then anonymized or deleted

**Gender & Vulnerability Considerations**:
- Explicit recruitment of women (47 cooperative members identified)
- Safe space for interviews (women surveyor with female respondents if preferred)
- Accessible language (Portuguese, not technical jargon)
- Attention to power dynamics (community leader not present during household interviews)

## Overview
Open-source framework for identifying optimal sites and technologies for 
community-scale solar systems in low-income regions. Case study: Huíla Province, Angola.

## Key Features
- MCDA-GIS integration (AHP-weighted overlay analysis)
- 6 criteria: solar irradiance, population, distance-to-grid, terrain, LULC, existing infrastructure
- Technology recommender (3 options: PV-fixed, PV-tracker, hybrid solar-diesel)
- Sensitivity analysis (42 scenarios; ranking stability confirmed)
- Reproducible: <30 min from data download to results

## Quick Start
```bash
git clone https://github.com/ISPTEC-Energy/geesp-angola.git
cd geesp-angola
docker build -t geesp:1.0 .
docker run -it geesp:1.0 jupyter notebook  # Launches interactive demo
```

## Results at a Glance
- **3 priority zones identified** in Huíla Province (45 communities, 191K people)
- **LCOE: USD 0.18–0.22/kWh** (vs diesel USD 0.40–0.45/kWh baseline)
- **Payback period: 6.7 years** (18 MV aggregate)
- **CO₂ avoided: 50K ton/year**
